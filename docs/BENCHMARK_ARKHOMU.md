## 2. 何が効いているか（構造分解）

1. **ビルド探求の深さが沼を作る**: 2,000スキルの組み合わせ理論値探し＝「攻略対象が他人の編成と無限のビルド空間」。プレイヤーが自分で遊びを生成する。500体は「多い」ではなく**ビルド素材の語彙数**。
2. **サーバー型が社会的ループを解放**: ランキング(競争)・フリマ(経済)・Wiki/Discord(知識共有)は全部アカウント制の上に載る。**単独プレイでは絶対に出ない持続力**がここから来る。
3. **放置要素が生活に馴染む**: 遠征=ログオフ中も進む→毎日戻る理由。社会人プレイヤーの時間と喧嘩しない。
4. **高速更新が「生きてるゲーム」感を作る**: 日次更新はそれ自体がコミュニティへのコンテンツ供給。
5. **コミュニティ流通を最初から設計**: 事前登録→β→正式のハイプ導線／ギフトコード枠（攻略サイトが記事を書く動機＝SEOフライホイール）／配信許可明記＋配信者募集（無料の宣伝部隊）。
6. **ポータル掲載が「同時接続248人」を可視化**: 数字の社会的証明がさらに人を呼ぶ。

## 3. アルケミスト・オーダーとの正直な差分

| 軸 | arkhomu | AO現状 | 評価 |
|---|---|---|---|
| ビルド深度 | 2,000スキル自由合成 | 技4枠+遺伝技+talent+持ち物 | **大差**。AOは「相性を突く」が深さの主軸で、ビルド語彙が少ない |
| 社会性 | PvP/フリマ/ランキング | なし(週塔v1もローカル) | **大差**。ただしF2/F5のロードマップは正しくこの方向を向いている |
| 放置 | 遠征・拠点 | デイリー討伐のみ | 差あり |
| 手軽さ | 要アカウント | **URLだけ・登録不要** | **AOが優位**。唯一明確に勝っている軸 |
| 演出/世界観 | (未確認・テキスト中心とみられる) | ドット絵100体・マップ演出・物語 | AOに分がある可能性 |
| 更新体制 | 日次 | 週次想定 | 差あり(だが時間ガードレール上、追わない) |

**結論: 「同じ土俵で殴り合わない」。** arkhomuはライブサービス型の沼ゲー。AOは「登録不要・3秒で始まる・物語と収集の手触り」のスナック型。ただし**retention構造（戻る理由・競争・知識共有）は借りるべき**で、それは既にF2(週塔)/F5(順位表)として設計済み＝**この路線の価値が実証された**と読む。

## 5. 借りないもの（明確に断る）

- **サーバー型への全面転換・フリマ/PvP経済**: 運用が生活を食う。REVENUE_PLANの時間ガードレール(週5h)と両立しない。F5の限定的順位表(再シミュ検証)までが上限。
- **日次更新体制**: 同上。週次リズム(月曜塔)で「生きてる感」を出すのがAOの分相応。
- **500体/2000スキル級の物量競争**: 物量はarkhomuの土俵。AOは手触り・物語・手軽さで違う客を取る。

## 6. ポジショニングの再確認（このベンチマークを受けて）

- arkhomu = 「沼に住む人のゲーム」（アカウント作って毎日3時間）
- **AO = 「URLを踏んだ3秒後に冒険が始まるゲーム」**（登録不要・スキマ時間・懐かしい手触り）
- 訴求文言に「登録不要」をこれまで以上に前置きする。arkhomuのヒットで「ブラウザ合成ゲー」に流入した層の**セカンドゲーム needs**（もっと軽いのも欲しい）を拾える位置に立つ。
- 競合ではなく**市場実証**と捉える: 2026年の日本に「ブラウザでモンスター合成」の現役需要が確実にある。

## 7. ゲーム内システム詳細分析（2026-07追補・攻略Wiki+実プレイ観察）

情報源: 攻略Wiki(施設ページ群)＋ユーザー実プレイの観察。ログインボーナス/スタミナはWiki未文書化のため実プレイ観察ベース。

### 7.1 経済の背骨=「アイテムが全活動の接着剤」(700種)
```
探索(スタミナ消費) → アイテムdrop 700種
   ├→ ホムンクルス生成所: 素材2つ+器 → 新ホム誕生
   ├→ アイテム錬金: 素材 → 上位アイテム/施設強化材
   ├→ 遠征の弁当: 食事アイテム → 派遣中のステ強化
   ├→ 装備(ハクスラ)
   └→ フリマ出品(経済)
```
**核心**: どの活動のdropも必ず別の活動の入力になる。「無駄なドロップが存在しない」から周回が退屈にならない。
アイテム総数(700)より**全アイテムに行き先がある設計**が本質。

### 7.2 ホムンクルス生成所（アイテム→モンスター生成）
- 素材2つ＋「器」(標準/強化/黄金/特別=生成値0-3)で生成。追加素材枠あり
- **素材が決めるもの**: ①特定ホムの出現トリガー ②素材レアリティ=RANK上限 ③ボーナスパラメータ・継承スキル
- 施設強化で「追加素材枠」「**予測数**(生成前に候補が見える)」が増える
- 楽しさの構造: **ドロップ=ガチャ券化**（拾った素材の組み合わせで狙い撃ち→予測で確かめ→回す）。計画と発見が同居

### 7.3 訓練所＋遠征（放置・時間装置）
- 訓練所: 枠4→8(拠点強化)、探索経験値の60→90%を**編成外にも**分配。「空き枠数がメニューに常時表示」=管理欲を刺激
- 遠征: 弁当を持たせて派遣(最短30分)、帰還時アイテム+弁当でステボーナス。派遣中は編成/訓練に使えない(機会費用)
- 効果: **ログオフ中も進む**＋「回収する用事」が数時間おきに発生=再訪問アポイントメント

### 7.4 ログインボーナス/スタミナ（実プレイ観察）
- ログボ: 日次報酬。コロシアム参加だけでも名誉Pt→スタミナ剤/クローバー交換=「毎日1タップの用事」を多重化
- スタミナ制: 探索の主資源。回復待ち/アイテム回復。**セッション区切り＋翌日の再訪理由**として機能
- 図鑑から生成/合成へ直接ジャンプできるUI(更新履歴より)=「欲しい」から「作る」への導線が1タップ

### 7.5 AOへの移植判断（結論）

| arkhomuの要素 | 移植判断 | AOでの形 |
|---|---|---|
| アイテム→モンスター生成 | **◎ 本命採用** | 「素材錬成」— 錬金テーマの完成形。詳細= [SPEC_ITEM_ALCHEMY.md](SPEC_ITEM_ALCHEMY.md) |
| 全dropに行き先がある素材経済 | **◎ 採用(小規模)** | 素材20-30種から開始。700種は追わない(物量は向こうの土俵) |
| ログインボーナス | **○ 採用(軽量)** | 「師匠からの小包」日次1タップ。連続縛りなし(罰でなく贈り物)。同spec§6 |
| 遠征(放置) | **○ 採用(軽量)** | 「派遣調査」— ボックスの遊休幻獣に仕事を与える。同spec§7 |
| 訓練所(編成外exp) | △ 保留 | 派遣調査が実質兼ねる。二重実装は過剰 |
| **スタミナ制** | **✕ 不採用** | 「好きなだけ遊べる」はAOのUSP。スタミナの効能(区切り/再訪理由)は錬金釜の完成待ち・ログボ・週塔で代替する |
| 図鑑→生成ジャンプUI | ○ 採用 | 図鑑に入手ヒント+錬成レシピへのリンク |

**スタミナ不採用の理由を明記**: スタミナは(a)回復課金 (b)サーバー負荷制御 (c)再訪アポの3機能。AOは(a)課金しない(b)サーバーない、なので価値は(c)のみ。(c)は「時間で完成する仕込み」（ポジティブな待ち）で代替できる。遊びたい人を止める仕組みは登録不要USPと正面衝突する。
