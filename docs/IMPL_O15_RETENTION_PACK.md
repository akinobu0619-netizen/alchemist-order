## 1. 捕獲チェーン（PokéFarm Q型・収集の沼化）

**仕様**: 同じ種を連続で捕獲するとボーナスが積み上がる。別の種を捕獲したらリセット（捕獲以外＝討伐/逃走では切れない）。

- `GameState` に `chain?: { speciesId: string; count: number }` を追加（types.ts。loadGameの互換補完も）
- **更新タイミング**: 捕獲成功時のみ。同種なら count+1、別種なら `{speciesId, count:1}` に置換
  - 更新箇所は Battle.tsx `throwFlask()` 成功ブロック と quickResolve.ts の捕獲成功ブロックの2箇所（state.ts に `applyCaptureChain(s, speciesId): GameState` ヘルパを作り両方から呼ぶこと。ロジック複製禁止）
- **効果**（次回以降、チェーン対象種と遭遇したときに適用）:
  | チェーン数 | 効果 |
  |---|---|
  | 3+ | その種の talent 下限2（rollTalent結果を `Math.max(2, roll)`） |
  | 5+ | talent 下限4 |
  | 7+ | 変異種率 1/100→**1/50** |
- 適用箇所: Battle.tsx `makeWild()`。config経由で渡す: Explore.tsx の battle config 生成時に `chain: state.chain` を同梱（BattleConfig拡張。**セーブには入れない**、configは揮発）
- **UI**:
  - 戦闘開始時、遭遇種がチェーン対象なら敵名の横に `🔗{count}` チップ
  - 捕獲成功の GetMonsterOverlay に「チェーン {n} 連鎖！」表示（n>=2のとき）
  - ホームの進捗カード枠に現在チェーン表示（`🔗 コグリフ ×4` など。チェーン無しなら非表示）
- 計測: `chain_update{species, count}`

## 2. 今日の幻獣（日替わりターゲット・毎日戻る理由）

**仕様**: 日付から決定的に1種が選ばれ、その日はその種にボーナス。

- `src/game/state.ts` に追加:
  ```ts
  export function speciesOfTheDay(dateStr: string): MonsterData
  // cyrb53等の既存ハッシュ(rng.tsのcyrb128流用可)で dateStr をハッシュ →
  // DEX のうち role!=='legendary' && stage<=2 の配列から index = hash % len で選択
  ```
  ※ `Math.random`禁止・日付文字列は既存 `today()` を使用（全プレイヤー同日同種になる=SNSで「今日は○○の日」と話題共有できる）
- **効果**（その日のみ）:
  - 捕獲成功率 +15pt（Battle.tsx の捕獲パネル表示と throwFlask 判定の**呼び出し側**で `catchChance(enemy) + (isToday ? 0.15 : 0)` を clamp(0..0.95)。engineのcatchChance本体は触らない）
  - その日初めて捕獲したら報酬: 💰300 + 上傷薬1（`daily` に `todayCatch?: boolean` を追加して1日1回制御）
- **UI**:
  - ホームの進捗カードに「今日の幻獣」カード（スプライト小+名前。未捕獲種なら金枠）。タップで図鑑該当ページへ
  - 探索中、対象種と遭遇したら敵名横に `☀今日` チップ
  - 図鑑の該当セルにハイライト
- 計測: `sotd_catch{species}`

## 3. 7日サイクル小包ログボ（既存ログボの置換強化）

**仕様**: SPEC_ITEM_ALCHEMY §6 準拠。既存 `applyDailyLogin`（連続日数×金+フラスコ）を**7日サイクルの累計制**に置換。

- `loginTotal: number` を GameState に追加（累計ログイン日数。連続が切れても減らない=罰なし）。既存 `loginStreak` は表示用に残してよい
- 報酬表（`(loginTotal-1) % 7` で引く。素材アイテムはO12未実装のため代替品で開始、O12後に差し替え）:
  | 日 | 報酬 |
  |---|---|
  | 1 | 傷薬×2 | 2 | フラスコ×2 | 3 | 💰300 | 4 | 上傷薬×1 |
  | 5 | フラスコ×3 | 6 | 全癒の秘薬×1 | 7 | **才能の結晶×1** |
- **UI**: 現在のダイアログ1枚を維持しつつ「師匠からの小包が届いている」の文言＋7枠の進捗ドット（今日が何日目か見える）を表示。菱形の演出は不要、既存モーダルで可
- grantReward を使うこと（Reward型は対応済み）。計測: `login_bonus{day}`

## 4. 帰還リザルト（セッションの締め・ポータル設計則）

**仕様**: 探索から「帰還する」時に、そのセッションの成果サマリを1枚出す。

- Explore.tsx にセッション集計 state: `{ battles: number; catches: {id,name,mutant,talent}[]; gold: number; events: number; newDex: number }`
  - 戦闘勝利/捕獲/宝箱/ゲル増分を加算（onStartBattleの結果はBattleから戻らないため、**探索入場時のstateスナップショットとの差分**で算出するのが簡単で確実: caught.length差分・money差分・wins差分）
- 「帰還する」押下時にモーダル表示:
  - 見出し「探索の成果」＋ 捕まえた幻獣のスプライト列（NEWは金枠・変異は✨）
  - 「図鑑 {caught}/{DEX_TOTAL}（+n）」「💰+{gold}」「🔗チェーン継続中: ○○×n」
  - ボタン: [拠点へ戻る]（既定） [もう一度潜る]（セッション継続の再訪導線）
- 成果ゼロ（差分なし）ならモーダルを出さず従来どおり即帰還
- 計測: `explore_summary{battles, catches, gold}`

## 5. タイプ別コンプ勲章（図鑑の中間ゴール）

**仕様**: 図鑑をタイプ別に見て、各タイプの捕獲率マイルストーンで報酬。

- 集計: `DEX.filter(m => m.type === t)` に対する `caught` の割合（9タイプ）
- 各タイプ 50% 到達で 💰500、**100% 到達で 才能の結晶×1**（`dexTypeClaimed?: string[]` で受取管理。'火50','火100' 形式）
- UI: Dexに「タイプ別達成」セクション（9行: タイプバッジ+プログレスバー+報酬受取ボタン）。記録タブにも合計表示1行
- 計測: `dex_type_complete{type, tier}`

## 6. 既存機能への追加指定（おまかせ遠征のゲート）

89520f5 の「おまかせ遠征」は**踏破済みコンテンツの周回ツール**に位置づける:
- 表示条件: そのノードで深度選択を1回以上経験済み（`progress.visited: string[]` にノードidを記録）または該当世界のボス撃破済み
- 未達ノードではボタン非表示（初見体験がワンクリックで飛ばされるのを防ぐ＝「観るバトル、選ぶ捕獲」の初回体験を守る）
- おまかせ遠征の結果ログに、§1チェーン・§2今日の幻獣のボーナスも正しく反映されること（quickResolve側にも同ヘルパを通す）
