# 実装プラン — アルケミスト・オーダー

## 推奨技術スタック（実験プロトタイプ向け）

**第一候補: React + TypeScript + Vite + Phaser（または独自Canvas）**

理由:
- データ駆動（`monsters.json` をそのまま読み込める）でClaudeと反復開発しやすい。
- まずブラウザで即動かせる → 後から **Capacitor** でiOS/Androidアプリ化、**Tauri** でデスクトップアプリ化が可能。「アプリで実装」要件を低リスクで満たせる。
- UI（メニュー/図鑑/バトルログ）はReact、フィールド探索やアニメはPhaser/Canvasで分担。

代替案:
- **Flutter (Dart)** … 最初から本格モバイルアプリにしたいなら最有力。ただしWeb即試遊性は劣る。
- **Unity** … リッチな2D/3Dを目指すなら。学習コスト高、Claudeとの反復はやや重い。

> 実験フェーズはまず「Web(React+TS)で遊べるMVP」を強く推奨。面白さが確認できてからネイティブ化。

## アーキテクチャ（データ駆動）

```
/data        monsters.json, moves.json, typechart.json, maps.json
/src
  /engine    battleEngine.ts (ダメージ計算/状態異常/相性), evolution.ts
  /state     セーブデータ(ローカルストレージ→後でクラウド)
  /ui        パーティ/図鑑/バトル/ショップ画面
  /scenes    フィールド探索, エンカウント
```

ゲームロジックを純粋関数（`battleEngine`）に閉じ込め、UIから分離 →
ユニットテストしやすく、バランス調整もJSON編集で完結。

## ダメージ計算式（初期案）

```
damage = floor( ((2*Lv/5 + 2) * 技威力 * ATK/DEF) / 50 + 2 )
         * 相性倍率 * タイプ一致(1.5) * 乱数(0.85〜1.0)
```

## 開発ロードマップ（MVP→拡張）

1. **MVP-1 バトルだけ** … 御三家3体 vs 野生1体、相性・進化なしの1対1ターン制。コア面白さ検証。
2. **MVP-2 育成ループ** … 経験値/レベル/進化/捕獲を実装。図鑑画面。
3. **MVP-3 フィールド** … 1マップ＋エンカウント＋最初のジム戦。
4. **拡張** … 全100体・ストーリー・ラスボス・分岐エンド。
5. **アプリ化** … CapacitorでAndroid/iOSビルド。

## アセット（幻獣の絵）

- 既に画像で図鑑風の絵がある → これを切り出して各幻獣のスプライトに。
- 量産は画像生成AIで「同一画風プロンプト」を固定して統一感を確保。

## まず作るなら
「MVP-1（1対1バトルが動くWebプロト）」から着手するのが最短。
`monsters.json` の御三家データはもう使える状態。
